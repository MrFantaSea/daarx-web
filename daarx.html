01 / SYSTEM BRIEF

Privacy is a boundary.

The design targets private value on a deterministic base. It does not turn a protocol into a blanket anonymity guarantee.

01

THE SUPPLY

Integer-only, then zero.

The current consensus design targets π × φ base units—about 5.083 septillion—reached exactly by an integer-only hundred-year schedule and then zero emission. The table is implemented and testable, but it is not a promise about an unlaunched asset and remains subject to final consensus review and the genesis process.

02

THE PRIVACY

Membership across the chain.

DAARX's current consensus implementation activates CTS full-chain membership proofs from genesis and exercises the real prover and verifier in local tests. Fees are designed to use a short public ladder to reduce fee fingerprinting.

03

THE LIMIT

Tested is not proven.

The privacy layer is new cryptography built on reviewed dependencies. The project has not completed the independent review and long public soak required for a launch claim. A security program is planned, but no bounty is currently open.

02 / PRIVACY INSPECTION

Mechanism. Evidence. Boundary.

LAYER 01 / CONSENSUS TARGET

The crowd is the cover.

CTS full-chain membership proofs are intended to let a valid spend prove membership without identifying the spent output. Hidden amounts and spent-tag checks remain part of deterministic consensus.

An anonymity-set design is not a guarantee of end-to-end anonymity.

LAYER 02 / CURRENT EVIDENCE

Real paths, local tests.

The current implementation exercises the real prover and verifier in local tests. Some upstream FCMP components have published review artifacts; DAARX's composition, adapters, and consensus integration have not received an independent professional security audit.

The ingress policy and end-to-end miner path still require launch-level integration evidence.

LAYER 03 / THREAT SURFACE

The ledger is not the whole system.

Metadata, endpoint compromise, implementation flaws, network observation, and novel cryptanalysis remain threats that require public testing. Protocol privacy cannot erase failures outside its proven boundary.

No independent professional security audit or long public soak has been completed.
